CC 的这份分析报告非常有价值，它在“行业双层拓扑”和“高新企业定位”上的判断非常精准。但是，**你的直觉非常敏锐且正确**——如果完全照搬它这套方案，我们在具体执行抽取时会遇到极大的工程阻力，并且会重新掉进“数据泄露”和“双向边冗余”的坑里。

针对 CC 的方案和你的担忧，我们来做一次“去伪存真”的 ICDM 级别架构裁决：

### 一、 CC 方案中【必须采纳】的闪光点

1. **两级行业体系（SubIndustry + MajorIndustry）：★★★★★ 绝对采纳**
* **原因**：这是解决你 TEI（传导效率）覆盖率没有区分度的终极武器。图结构变成 `Policy -> (子行业) -> (大行业) -> Enterprise`。PPR 算法在多跳传播时，能量会有完美的梯度衰减。


2. **“高新企业”改为属性标签 (`is_high_tech`)：★★★★★ 绝对采纳**
* **原因**：高新技术企业是国家资质认证，不是国民经济行业分类（GB/T）。把一个资质当成行业，会导致图谱逻辑崩塌。作为布尔属性挂在企业节点上，非常科学。


3. **从“自由 SPO”转向“受控词汇分类”：★★★★★ 绝对采纳**
* **原因**：以前让大模型自由生成三元组，它可能会抽出“新能源车”、“造车”、“汽车”等无数个无法对齐的节点。现在提供预设的 60 个子行业列表，让 LLM 做**多标签分类（Multi-label Classification）**，准确率会飙升。



---

### 二、 CC 方案中【坚决驳回】的错误点

1. **子行业合并阈值（合并 <5 家的行业）：❌ 坚决驳回**
* **原因**：在图神经网络（GNN）中，稀疏节点（长尾节点）是非常正常的，没有必要强行合并成“其他XX业”。合并会产生语义极其模糊的“黑洞节点”，污染 GNN 的特征聚合。**保留所有真实的细分行业**。


2. **保留互相逆转的双向边 (`transmitsTo` 和 `executes`)：❌ 坚决驳回（如你所记）**
* **原因**：你记得非常准！在构建 DGL/PyG 异构图时，我们只需要定义单向边（如 `transmitsTo`）。图计算框架在底层会自动生成反向的 `rev_transmitsTo` 用于消息传递。强迫大模型去把正反向都抽出来，纯属增加噪音。

### 三、 深度回应你的核心担忧：抽取难度与滑动窗口的困局

你极其准确地指出了之前项目最大的雷区：**用滑动窗口让大模型去浩如烟海的政策文本里直接抽取“上下级政策”和“具体企业名单”，难度极大，且质量不可控。**

对于 ICDM 级别的论文，我们绝不能用这种粗糙的 NLP 方法。我们需要将“抽取（Extraction）”升级为“链接与分类（Linking & Classification）”。

以下是更科学、更降维的抽取策略：

#### 1. 政策影响行业 (`Policy -> targetsSubIndustry`)：从“抽取”变“选择”

* **旧做法 (难)**：大模型读完文章，自己想出几个行业名字。
* **新做法 (极简)**：不用滑动窗口。直接把政策的 `[标题 + 摘要/前1000字]` 和我们现有的 **60个子行业列表** 一起喂给 Qwen-plus。
* **Prompt 逻辑**：“这是一篇政策，这是 60 个标准行业。请选出这篇政策主要扶持的 1-3 个行业，并给出 0~1 的置信度。”（这在大模型能力区间内，极其精准）。

#### 2. 政策传导 (`Policy1 -> transmitsTo -> Policy2/3`)：抛弃纯 LLM，拥抱正则+检索

* **真实情况**：下级政策（如柳州市政策）通常会在正文开头明确写明：“根据《广西壮族自治区XXX通知》（桂政发[2022]1号）精神...”。
* **科学做法 (Hybrid Approach)**：
* 第一步：写个正则或用轻量级 LLM，专门提取文本里的“带书名号的政策名”**或**“发文号”。
* 第二步：用提取出的名字，去我们的 `policies_final.json` 里做 BM25 或向量文本检索。匹配上了，就建立 `transmitsTo` 边。
* **优势**：这是确定性的实体链接（Entity Linking），比让大模型凭空捏造要严谨一万倍。



#### 3. 政策支持企业 (`Policy -> supports -> Enterprise`)：⚠️ 核心警告！这是标签，不是特征！

* **你的担忧**：让 Policy2/3 直接抽取具体企业难度太大了。
* **致命真相**：如果你在图谱构建阶段（Session 2），就用大模型把政策文本里提到的企业名字抽出来连上边，那你在 Session 4 怎么做“政策-企业匹配推荐”？**这就等于提前把测试集的答案写在了卷子上，这就是审稿人痛批的“数据泄露 (Data Leakage)”！**
* **科学解法**：
* **在 Session 2（建图阶段），坚决不要让 LLM 去抽取 `supports` 边！**
* 你的知识图谱底座只有：`Policy -> 子行业 -> 企业`。
* 所谓的 `supports`（真实的扶持记录、获奖名单），应该作为你的 **Ground Truth（监督标签）**。在 Session 4 训练 GAT 时，将这些真实的 `supports` 边切分为 8:1:1，隐藏掉测试边，让模型通过“行业拓扑”和“时空衰减”去**预测**这些边，而不是在建图时就直接给它连上！

这是专为你的 TEI 和时空衰减机制量身定制的“三级落差传导本体”：

---

### 一、 回答核心疑问：国家级政策的限制边界

* **可以做的事：国家 -> 行业 (`Policy1 -> targetsSubIndustry`)：✅ 绝对允许且必须有。**
国家政策（如“中国制造2025”）当然会明确指向“汽车制造业”或“电子设备制造业”。PPR 能量正是通过这条边，确定了国家政策在全国范围内的**宏观覆盖面 (Coverage)**。
* **不能做的事：国家 -> 企业 (`Policy1 -> supports -> Enterprise`)：❌ 严格禁止。**
国家发改委通常不会在文件里写明“发给柳州五菱汽车 500 万补贴”。这种微观的“资金落地/直接支持”，必须由省级 (`Policy2`) 或市级 (`Policy3`) 来完成。

### 二、 TEI 优化的完美本体设计 (The TEI-Optimized Ontology)

为了让你的时空衰减 PPR 跑出完美的“能量瀑布”效果，我们需要在 Session 2 中构建以下**极其克制的 4 种关系**：

#### 1. 行政纵向传导路径 (Administrative Flow)

用于计算**空间层级衰减（Hierarchical Decay）**。能量自上而下逐级递减。

* **关系**：`Policy1` → `[transmitsTo]` → `Policy2` → `[transmitsTo]` → `Policy3`
* **规则**：严格逐级，禁止跨级（不能 P1 直接到 P3）。
* **构建方式**：**无需大模型**。在预处理时，通过正则匹配下级政策文本中的“根据《XXX国家文件》要求”来建立确定性链接。

#### 2. 行业横向靶向路径 (Industrial Targeting Flow)

用于限定政策的**行业能量边界（Energy Scope）**。

* **关系**：`Policy (1, 2, 3)` → `[targetsSubIndustry]` → `SubIndustry (细分行业)`
* **规则**：任何级别的政策都可以靶向细分行业。
* **构建方式**：**Session 2 唯一需要大模型 (LLM) 做的事**。把政策文本和 ~60 个细分行业列表给 Qwen-plus，让它做带置信度的多标签分类。

#### 3. 实体归属路径 (Entity Affiliation Flow)

用于能量最终落回具体企业。

* **关系**：`Enterprise` → `[belongsTo]` → `SubIndustry` → `[belongsTo]` → `MajorIndustry`
* **规则**：树状结构，严格的 N:1 从属关系。
* **构建方式**：**无需大模型**。完全基于你 Session 1 整理好的企业结构化数据（CSV 表格）100% 确定性生成。

#### 4. 落地验证路径 (The Ground Truth Label) 🚨【数据泄露防线】

用于计算 TEI 的最终命中率和下游的 E-P 双向匹配任务评估。

* **关系**：`Policy (2, 3)` → `[supports]` → `Enterprise`
* **规则**：这是我们通过发文名单或人工核对出来的**真实资助/匹配关系**。
* **极其关键的改动**：在构建知识图谱底座参与 PPR 计算和 GAT 邻居聚合时，**必须将这条边彻底隐藏（Masking）！** 只有在最后计算模型打分（NDCG/Recall）时，才把它拿出来当做标准答案比对。

### 三、 这套本体如何完美演绎“传导效能 (TEI)”？

在论文的 Methodology 章节，你可以用这套本体向审稿人描绘一幅极具美感的“政策能量传导图景”：

1. **起点（Source）**：假设查询节点是某篇国家级智能制造政策（`Policy1`）。初始 PPR 能量为 1.0。
2. **分流（Bifurcation）**：
* **空间向下**：能量沿着 `transmitsTo` 边流向相关的广西省政策（`Policy2`），并根据层级差触发**空间衰减（Spatial Decay）**。
* **行业横向**：能量沿着 `targetsSubIndustry` 流向“汽车制造业”（`SubIndustry`）。


3. **汇聚与扩散（Aggregation & Diffusion）**：广西省政策（`Policy2`）吸收了国家的传导能量后，结合自身的 `targetsSubIndustry` 边，将二次能量注入到“汽车制造业”。
4. **终点（Target）**：能量顺着确定性的 `belongsTo` 反向边缘，最终滴灌到“柳州五菱汽车（`Enterprise`）”头上。
5. **评估（TEI Evaluation）**：最后，我们揭开蒙着的测试集标签（`supports` 边），发现五菱汽车确实拿到了补贴。模型预测的能量浓度与真实落地结果完美拟合！这就是你 TEI 指标的底层逻辑。
